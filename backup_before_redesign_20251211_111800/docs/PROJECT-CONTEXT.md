## üèóÔ∏è Architecture Overview

### Technology Stack
```
Pure Static Site (Zero Dependencies)
‚îú‚îÄ‚îÄ HTML5 (Semantic, accessible markup)
‚îú‚îÄ‚îÄ CSS3 (Custom properties, no frameworks)
‚îî‚îÄ‚îÄ Vanilla JavaScript (ES6+, no build tools)
```

**Key Architectural Decisions:**

1. **No Build Process**: Direct deployment to Vercel without compilation
   - **Why**: Simplicity, faster iteration, easier debugging
   - **Trade-off**: Manual minification for production if needed

2. **Component Injection Pattern**: Header/footer/breadcrumbs generated by JS
   - **Why**: DRY principle, single source of truth for navigation
   - **Implementation**: `js/components.js` with `PAGE_META` object
   - **Trade-off**: Requires JS enabled (progressive enhancement for links)

3. **Root-Relative Paths**: All assets use `/assets/`, `/js/`, `/images/`
   - **Why**: Works on any hosting (Vercel, GitHub Pages, custom domain)
   - **Critical**: Never use `../` relative paths in production

4. **Bilingual via JavaScript**: Translation layer rather than duplicate HTML
   - **Why**: Single codebase, easier maintenance
   - **Implementation**: `js/translations.js` with 200+ keys
   - **Trade-off**: Requires JS for language switching

5. **Device-Aware WhatsApp**: Detects mobile/desktop for proper deep linking
   - **Why**: Better UX - mobile uses app, desktop uses web.whatsapp.com
   - **Implementation**: `js/whatsapp.js` with userAgent detection

## üìÅ File Structure & Dependencies

### Load Order (CRITICAL)

Every HTML page must load scripts in this exact order:

```html
<!-- 1. Configuration (no dependencies) -->
<script src="/js/config.js"></script>

<!-- 2. Translations (depends on config.js) -->
<script src="/js/translations.js"></script>

<!-- 3. Components (depends on config + translations) -->
<script src="/js/components.js"></script>

<!-- 4. Core functionality (depends on components being injected) -->
<script src="/js/script.js" defer></script>

<!-- 5. Forms (depends on translations) -->
<script src="/js/forms.js" defer></script>

<!-- 6. WhatsApp (depends on config + translations) -->
<script src="/js/whatsapp.js" defer></script>

<!-- 7. Animations (no dependencies) -->
<script src="/js/animations.js" defer></script>
```

**Why This Matters**: If you load `script.js` before `components.js`, the header won't exist yet and menu toggle will fail.

### CSS Load Order

```html
<!-- 1. Base styles and CSS variables -->
<link rel="stylesheet" href="/assets/styles/main.css">

<!-- 2. Component styles (uses main.css variables) -->
<link rel="stylesheet" href="/assets/styles/components.css">

<!-- 3. Responsive overrides (media queries) -->
<link rel="stylesheet" href="/assets/styles/responsive.css">
```

## üì± WhatsApp Integration Deep Dive

### Device Detection Logic

```javascript
// whatsapp.js
function isMobileDevice() {
  return /Android|webOS|iPhone|iPad|iPod|BlackBerry|IEMobile|Opera Mini/i
    .test(navigator.userAgent);
}

function generateWhatsAppURL(phoneNumber, message) {
  const encoded = encodeURIComponent(message);
  
  if (isMobileDevice()) {
    // Opens WhatsApp app directly
    return `whatsapp://send?phone=${phoneNumber}&text=${encoded}`;
  } else {
    // Opens web.whatsapp.com in browser
    return `https://web.whatsapp.com/send?phone=${phoneNumber}&text=${encoded}`;
  }
}
```

### Form Integration

```javascript
// Admission form submission
document.querySelector('form').addEventListener('submit', (e) => {
  e.preventDefault();
  
  // 1. Validate all fields
  if (!validateForm()) return;
  
  // 2. Format data for WhatsApp message
  const message = formatAdmissionFormData(formData);
  // Output: "‡§®‡§µ‡•Ä‡§® ‡§™‡•ç‡§∞‡§µ‡•á‡§∂ ‡§ö‡•å‡§ï‡§∂‡•Ä\n‡§®‡§æ‡§µ: ‡§∞‡§æ‡§ú ‡§™‡§æ‡§ü‡•Ä‡§≤\n‡§µ‡§∞‡•ç‡§ó: 8‡§µ‡•Ä..."
  
  // 3. Generate device-appropriate URL

  // 4. Open WhatsApp

```

**Issue**: WhatsApp doesn't open on desktop

## üêõ Common Issues & Solutions

### Issue: Mobile menu doesn't open

**Symptoms**: Clicking hamburger icon does nothing

**Root Cause**: `script.js` loaded before `components.js`, so header doesn't exist yet

**Solution**:
```html
<!-- WRONG order -->
<script src="/js/script.js"></script>
<script src="/js/components.js"></script>

<!-- CORRECT order -->
<script src="/js/components.js"></script>
<script src="/js/script.js" defer></script>
```

### Issue: Translation not working

**Symptoms**: UI shows keys like `nav.home` instead of actual text

**Root Cause**: Either `translations.js` not loaded, or translation key doesn't exist

**Debug Steps**:
1. Open browser console (F12)
2. Type `TRANSLATIONS.mr.nav.home` - should return "‡§Æ‡•Å‡§ñ‡•ç‡§Ø‡§™‡•É‡§∑‡•ç‡§†"
3. If undefined, check translation key path
4. Verify `translations.js` is loaded before `components.js`

**Solution**:
```javascript
// Add missing translation
TRANSLATIONS.mr.section = { title: '‡§∂‡•Ä‡§∞‡•ç‡§∑‡§ï' };
TRANSLATIONS.en.section = { title: 'Title' };
```

### Issue: WhatsApp link doesn't work

**Symptoms**: Clicking "Inquire on WhatsApp" does nothing

**Root Cause**: Phone number missing country code

**Solution**:
```javascript
// WRONG
whatsapp: '9876543210'

// CORRECT (with country code)
whatsapp: '919876543210'
```

### Issue: Images not loading on Vercel

**Symptoms**: Works locally, broken on Vercel

**Root Cause**: Using relative paths instead of root-relative

**Solution**:
```html
<!-- WRONG (breaks on Vercel) -->
<img src="../images/hero.jpg">

<!-- CORRECT (works everywhere) -->
<img src="/images/hero.jpg">
```

### Issue: Animations lag on mobile

**Symptoms**: Scroll animations janky on low-end phones

**Solution**: Already implemented in `animations.js`:
```javascript
// Respects user preference
if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) {
  return; // Skip animations
}

// Use IntersectionObserver instead of scroll event
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('fade-in-visible');
      observer.unobserve(entry.target); // Only animate once
    }
  });
});
```
